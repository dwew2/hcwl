CYMRAEG

Regularisations

Two kinds of additional information are provided for each text: bibliographical, linguistic and other background information about the texts, and encoding of regularisation tags which can be used in XSLT output files designed for the use with concordance programs.

Background information on the texts

A brief introduction is provided to each text, with sections on the manuscript or print in which it is found, its author and/or scribe (if known), and a brief outline of its chief linguistic and orthographic characteristics and geographical ('dialectal') provenance.

This brief introduction aims to give users the basic textual information that they will need to use and evaluate the text as a source, but extensive background research is not possible. The sections on language and orthography are intended to make it easier for the user to search a given text, and to warn of possible pitfalls.

Some of this information, particularly that on the geographical provenance of a text is necessarily rather tentative, and it is hoped that inclusion of a text in the corpus will encourage further reseach on its literary and philological background. The introduction also provides references to secondary material on a particular text, where this exists.

Regularisations

TEI provides for the encoding of regularizations where the original text exhibits a widely diverging spelling. This is of special interest when searching a given text by computer, for instance using a concordance program. The text itself always represents the form found in the source. Regularisations are encoded in the XML file using the orig element, with the regularised form being given in the reg attribute. For instance, if the text contained the form sane ‘socks’, which diverges significantly from present-day literary Welsh hosanau, a regularisation tag would be added as follows:

<orig reg="hosanau">sane</orig>

The texts originate from a period of over three hundred years and are written using various different spelling systems. All regularisations had to be added manually. The number of regularisations that were provided therefore had to be restricted, otherwise every other word would have a regularisation. Orthographic variants that can easily be searched for using regular expressions (regexp) were therefore not tagged using the <orig reg=""></P> tag.

Regularised forms normally correspond to those found in present-day literary Welsh. However, where the morphology of a form is itself obsolete in present-day Welsh, the regularisation may represent the form of an item as it is generally found in early modern literary Welsh.

Once the possibilities of regexp are taken into consideration, many of the common spelling variations become unproblematic, meaning that in such cases a regularisation in the TEI files is unnecessary.

The following spelling variation, which appears extensively throughout the corpus texts, is therefore not tagged explicitly, and users should take into account the possibility of such variation when using the corpus:
